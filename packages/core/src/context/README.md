# context 管理

## 上下文压缩

### 触发时机

主进程在每次发送用户消息前估算系统提示词、已有摘要、会话历史和本轮输入的 token 占用。
当估算值超过模型可用上下文的安全阈值，或继续发送会导致上下文不足时，压缩较早历史。

### 摘要与索引

提示词：

摘要输出固定为 Markdown，包含历史摘要、关键决策与事实、已完成操作、待跟进事项和索引。
摘要只作为系统上下文注入，不写入普通会话消息。

### 渐进式披露

提示词：

## user.md

### 触发时机

## AGENTS.md

### 触发时机

## System prompt

user.md + AGENTS.md + 历史会话索引 + 最近会话记录（5 group）

实现边界：

- 较早历史由压缩摘要提供。
- 最近 3-5 轮完整消息继续作为 Agent 历史传入。
- 用户可见聊天历史仍保留原始 `session_messages`。
