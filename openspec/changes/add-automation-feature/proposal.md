## Why

当前桌面应用只有手动进入工作空间并主动发起会话的能力，缺少可持续运行的自动化机制，无法把固定提示词在指定工作空间中按计划执行。现在需要先建立自动化的基础模型和定时触发能力，为后续接入更多触发来源（例如邮件）提供统一扩展点。

## What Changes

- 新增自动化数据模型，用于保存自动化名称、目标工作空间、自定义提示词、启用状态、触发器类型与触发配置、最近计划执行时间和最近实际执行结果等信息。
- 新增可扩展的触发器抽象，本期先支持基于 `node-cron` 的定时任务触发，并使用标准 cron 表达式定义计划，明确后续可以新增邮件等其他触发器类型而不破坏已有自动化记录结构。
- 新增自动化执行调度能力，使用 `node-cron` 在应用进程存活期间按计划触发自动化，并在应用重新启动后补偿最近一次错过的计划执行；如果错过多次，仅补最后一次。
- 新增自动化页面，展示已创建的自动化列表；当列表为空时提供空态引导创建入口，并在页面右上角提供“添加自动化”按钮。
- 新增自动化创建与编辑交互，允许用户选择一个工作空间并填写一段自定义提示词；定时配置支持“每天 xx 点”“每小时”“每周 周几 xx 点”“自定义 cron 表达式”四种方式，页面 UI 使用 shadcn-vue 组件实现。
- 新增面向 AI 的自动化管理工具，让 Agent 可以查询、更新、删除现有自动化，并在处理创建请求时先生成结构化预览草稿，而不是直接落库。
- 新增聊天中的自动化预览卡片交互：当 AI 生成待创建草稿后，界面展示时间、标题、摘要等预览信息；用户点击“创建”后拉起现有自动化表单，并将 AI 生成的内容作为默认值带入。

## Capabilities

### New Capabilities

- `automation-management`: 管理自动化记录的创建、查询、展示与基础配置，包括工作空间选择、自定义提示词、启用状态和空态引导。
- `automation-trigger-execution`: 定义可扩展触发器模型并执行自动化任务，本期支持定时触发、进程恢复后的最近一次遗漏补偿执行，以及后续扩展其他触发器类型。

## Impact

- `app/work/src/main/db/`：新增自动化相关表结构与迁移。
- `app/work/src/main/service/`、`app/work/src/main/controllers/`、`app/work/src/preload/`、`app/work/src/shared/`：新增自动化 CRUD、触发器配置和调度执行相关服务与 IPC 契约。
- `app/work/src/renderer/src/pages/auto/` 及相关 store/router：新增自动化列表页、空态页和创建入口，使用现有 shadcn-vue 组件体系实现。
- `packages/core/src/tools/`、会话工具渲染链路与聊天相关 renderer：新增自动化管理工具、预览卡片结果协议和“从卡片进入表单确认创建”的交互。
- 应用启动流程：需要在主进程初始化时恢复自动化调度，并处理最近一次遗漏执行的补偿逻辑。
