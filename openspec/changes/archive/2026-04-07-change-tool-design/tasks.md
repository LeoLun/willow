## 2. 权限控制与执行流程

- [x] 2.1 实现工具审批协调器，拦截需要审批的工具调用，按 `toolCallId` 存储待审批状态，并根据用户决策恢复或拒绝执行。
- [x] 2.2 将审批决策范围限制为当前单次工具调用，不实现会话级或持久化授权记忆。
- [x] 2.3 扩展主进程会话流和共享消息/状态类型，将待审批、已批准、已拒绝的工具调用状态同步到 renderer。
- [x] 2.4 为现有工具定义首版权限策略映射，确保 `ls`、`read`、`webfetch`、`websearch` 等低风险工具默认直通。
- [x] 2.5 为 `bash` 实现高危命令识别逻辑，并保证只有命中高危规则的命令才进入人工审批流程。
- [x] 2.6 保证受保护工具在被拒绝时返回明确的拒绝结果，而不是静默执行。

## 3. 前端审批体验

- [x] 3.1 更新 renderer 侧会话状态处理逻辑，在现有工具执行状态之外，额外跟踪待审批请求和已处理审批结果。
- [x] 3.2 改造工具展示组件/renderers，使其能够展示审批上下文、等待状态以及批准/拒绝操作。

- [x] 3.4 打通 renderer 到主进程的交互链路，使用户可以直接在聊天 UI 中处理待审批工具调用。
- [x] 3.5 确保审批 UI 只展示单次批准/拒绝操作，不展示会话记忆或持久化授权选项。
