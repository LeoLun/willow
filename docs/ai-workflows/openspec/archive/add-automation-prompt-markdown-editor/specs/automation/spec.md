# 自动化规格增量：Markdown Prompt 编辑器

## 修改需求

### 需求：支持在详情页以 Markdown 编辑自动化提示词

系统必须通过独立 workspace 包 `@willow/editor-md` 在自动化详情页中提供支持 Markdown 渲染与输入快捷转换的提示词编辑器，并继续以 Markdown 文本保存自动化 prompt。

#### 场景：提供独立 Markdown 编辑器包

- **前提** 开发者安装或构建 workspace
- **当** 解析 `packages/editor-md`
- **那么** 该包的 package name 为 `@willow/editor-md`
- **并且** 该包导出可被 Vue renderer 使用的 Markdown 编辑器组件
- **并且** 该包不依赖自动化业务 store、router 或 Electron API

#### 场景：使用 Markdown 编辑器替代普通文本域

- **前提** 用户打开某条自动化详情页
- **当** 页面渲染提示词编辑区
- **那么** 系统使用来自 `@willow/editor-md` 且基于 `@tiptap/vue-3` 的编辑器承载提示词内容
- **并且** 页面不再使用普通 `Textarea` 作为该提示词字段的主要输入控件
- **并且** 编辑器保持详情页主内容区的工作台视觉层级

#### 场景：Markdown 快捷输入转换为富文本结构

- **前提** 用户正在自动化详情页编辑提示词
- **当** 用户在空段落输入 `# ` 后继续输入文字
- **那么** 当前段落转换为一级标题
- **并且** 用户输入 `## ` 到 `###### ` 时可转换为对应级别标题
- **并且** 用户输入列表、引用或代码块的常见 Markdown 前缀时，编辑器将其转换为对应结构

#### 场景：打开已保存 Markdown 提示词

- **前提** 某条自动化的提示词包含 Markdown 结构
- **当** 用户打开该自动化详情页
- **那么** 编辑器按支持的 Markdown 结构渲染内容
- **并且** 标题、列表、引用和代码块等结构具有可读的视觉层级
- **并且** 不支持的 Markdown 片段至少以可读文本形式保留

#### 场景：保存 Markdown 文本

- **前提** 用户在自动化详情页编辑了提示词
- **当** 用户点击保存
- **那么** 系统提交 Markdown 文本字符串作为自动化 prompt
- **并且** 提交内容不包含 HTML 标签
- **并且** 提交内容不包含 Tiptap JSON 或 ProseMirror document 数据

#### 场景：编辑器参与现有详情页状态

- **前提** 用户正在自动化详情页编辑提示词
- **当** 用户修改、重置、保存或保存失败
- **那么** 编辑器内容与页面现有脏状态、重置逻辑、必填校验和保存反馈保持一致
- **并且** 保存失败不会丢弃用户尚未成功提交的编辑内容
