# 需求规范：应用内自动更新

## 需求

### R1: 检测与分类机制
- 应用启动时或用户在设置页面点击“检查更新”时，主进程 MUST 发送请求至 GitHub Release API：`GET https://api.github.com/repos/LeoLun/willow/releases/latest`。
- 系统 MUST 将返回的最新 Tag（如 `v1.0.1`）与当前应用的版本（通过 `app.getVersion()` 获取）进行对比。
- 版本对比必须遵循 SemVer 规范。若最新版本大于当前版本，则判定有新版本。
- 系统 MUST 区分更新类型：
  - **整包更新 (Full Update)**：若主版本号 (Major) 发生变更（例如 `1.0.0` -> `2.0.0`），或最新 Release 的 assets 中未提供增量包（即 `app.asar`），则判定为整包更新。
  - **增量更新 (Incremental Update)**：若主版本号未变且次版本号 (Minor) 或修订版本号 (Patch) 发生变更（例如 `1.0.0` -> `1.0.1`），且最新 Release 的 assets 中提供了 `app.asar` 文件，则判定为增量更新。

### R2: 架构匹配与资源匹配
- 判定有新版本后，系统 MUST 遍历最新 Release 的 assets 列表，筛选出符合更新类型的资产。
  - **整包更新 (Full Update)**：
    - 当前平台为 macOS (darwin)。
    - 若当前运行在 Apple Silicon 芯片（`process.arch === 'arm64'`），则匹配文件名包含 `arm64` 且以 `.dmg` 结尾的资产。
    - 若当前运行在 Intel 芯片（`process.arch === 'x64'`），则匹配文件名包含 `x64` 且以 `.dmg` 结尾的资产（若没有 `x64` 且无 `arm64` 字样的 dmg 也可以作为兜底）。
  - **增量更新 (Incremental Update)**：
    - 匹配文件名为 `app.asar` 的资产（或 `app.asar` 开头的 ASAR 文件）。

### R3: 下载流程与进度回显
- 用户确认更新后，主进程 MUST 开始下载筛选出的安装包资产（`.dmg` 或 `app.asar`），并将其保存到系统的临时目录中。
- 下载过程中，主进程 MUST 定期通过 IPC 发送下载进度（已下载字节数、总字节数、百分比）给渲染进程。
- 下载必须支持错误捕获，在下载失败或网络超时时通知用户，并允许重新尝试。

### R4: 安装与引导
- **整包更新 (Full Update)**：
  - 下载完成后，系统 MUST 提示用户。
  - 主进程 MUST 通过 Electron `shell.openPath(filePath)` 打开下载好的 `.dmg` 文件。
  - 系统 MUST 提示用户可以通过拖拽覆盖的方式，将新版本应用拖入 Applications 目录中完成更新。
- **增量更新 (Incremental Update)**：
  - 下载完成后，系统 MUST 提示用户“点击重启并更新”。
  - 用户触发更新后，主进程 MUST：
    - 定位当前运行的 `app.asar` 文件（位于 `process.resourcesPath/app.asar`）。
    - 将当前的 `app.asar` 重命名为 `app.asar.old`（用作备份和防止文件锁）。
    - 将下载好的新 `app.asar` 写入/重命名为 `app.asar` 路径。
    - 调用 Electron `app.relaunch()` 重启应用，并执行 `app.exit(0)` 退出当前实例。
    - 在应用下一次启动时，由背景任务异步检测并删除残留的 `app.asar.old` 文件。

### R5: UI 展现
- 在设置页面中添加一个更新状态管理区域：
  - 显示“当前版本：vX.Y.Z”。
  - 提供“检查更新”按钮。
  - 检查中状态：展示 Loading。
  - 已是最新状态：提示“当前已是最新版本”。
  - 有新版本状态：展示新版本号、更新类型（“大版本整包更新”或“增量热更新”）、发布日志（从 Release 的 `body` 字段获取，处理为纯文本或简易 markdown 展现）、以及更新操作按钮。
    - 若为**整包更新**，操作按钮为“下载更新安装包”。
    - 若为**增量更新**，操作按钮为“立即更新”。
  - 下载中状态：展示进度条与百分比。
  - 下载完成状态：
    - 若为**整包更新**，按钮变为“点击打开 DMG”（触发打开 DMG）。
    - 若为**增量更新**，按钮变为“重启并更新”。

## 验收标准

- [ ] 点击“检查更新”按钮时，触发 API 请求。
- [ ] 伪造本地版本（如修改 `package.json` 中的 `version` 降低），验证能够正确识别 GitHub 上的最新版本并弹出更新提示。
- [ ] 弹出更新提示时，能正确提取并显示 GitHub Release 页面中的更新日志（Markdown 格式），并显示正确的更新类型。
- [ ] 增量更新时，点击“立即更新”开始下载，界面能够实时看到进度条从 0% 增长到 100%。下载完成后，点击“重启并更新”，应用自动退出重启，且重启后版本号已升级至最新。
- [ ] 整包更新时，下载完成后，点击“点击打开 DMG”，能够自动挂载/打开 `.dmg` 卷。
