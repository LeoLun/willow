# Monaco Editor 自动折行规范

## 需求说明

### R1: 自动折行行为
- 当内联文件预览器（InlineFileViewer）加载的文件中包含超长单行文本，且该行宽度大于 Monaco Editor 容器的宽度时，编辑器 **MUST** 自动在容器边界处进行折行显示。
- 折行后，超出的文本应展示在下一行，且行号（Line Numbers）不应递增，仅作为逻辑行的延续。

### R2: 隐藏横向滚动条
- 在编辑器容器宽度不足以完整显示某些文本行时，编辑器 **MUST NOT** 出现横向滚动条，且用户 **MUST NOT** 能够通过鼠标滚轮或触控板进行横向滚动。
- 通过配置 `scrollbar: { horizontal: "hidden" }` 来隐式或显式禁用横向滚动条的渲染与交互。

## 验收测试用例

- [ ] **TC-1: 长文本自动折行验证**
  - 在侧边栏文件树中选择一个包含超长单行的文件（如 JSON 或 Markdown）。
  - 观察 Monaco Editor 渲染，超长单行应该在边缘处自动折断并换行显示。
  - 验证逻辑行行号（左侧 gutter）只在真实的新行上递增，折行部分不显示新的行号。

- [ ] **TC-2: 横向滚动条完全消失验证**
  - 调整侧边栏宽度，使其变得很窄。
  - 观察编辑器底部，没有任何横向滚动条出现。
  - 尝试在编辑器内进行左右拖拽或使用触控板横向滚动，确认无法发生任何水平位移。
