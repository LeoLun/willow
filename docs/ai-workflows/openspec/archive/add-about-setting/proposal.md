# 关于页面与系统更新迁移

## 动机

当前系统更新检查功能放置在“配置”设置页面。然而，“配置”页面主要承载模型密钥管理、网络搜索密钥管理等与应用第三方服务配置相关的内容，将“系统更新”放在此处不够直观。
此外，应用需要一个专门的“关于”页面来向用户展示应用的基础信息（如 Logo、名称、当前版本和应用简介），并提供系统更新的入口，这符合大多数桌面软件的用户习惯。

## 目标

1. 在设置页面的左侧导航栏中增加「关于」菜单项。
2. 创建「关于」设置页面，展示 Willow 应用的基本信息（应用图标、名称、当前版本号、应用简介）。
3. 将现有的「系统更新」模块（包含检查更新、更新下载进度、重启更新、大版本 DMG 提示、更新日志等 UI 与逻辑）从“配置”设置页面中完整迁移到新创建的「关于」设置页面。
4. 确保在「关于」页面中系统更新功能的所有数据流、IPC 通信与事件监听能够正常运行，并且「配置」页面完全移除系统更新相关残留。

## 范围

- **设置页面导航**：在 `Setting.vue` 的 `navItems` 中增加「关于」选项，指向新的路由。
- **路由注册**：在 `router.ts` 中为 `/setting` 注册子路由 `about`，并指向新页面 `AboutSetting.vue`。
- **关于页面创建**：在 `pages/setting/` 目录下新建 `about/AboutSetting.vue`，实现页面布局，展示应用信息与系统更新模块。
- **配置页面清理**：删除 `ConfigurationSetting.vue` 中所有与系统更新相关的 HTML 结构、事件监听、依赖的 icon 引入和方法。

## 非范围

- 不修改 Electron 主进程中的自动更新/更新检查机制（如 `UpdateService`、`CheckUpdateController` 等后端逻辑）。
- 不调整现有的自动更新 IPC 通信接口与协议。
- 不添加额外的用户关于信息（如最终用户许可协议 EULA、详细的开源许可证列表等）。
