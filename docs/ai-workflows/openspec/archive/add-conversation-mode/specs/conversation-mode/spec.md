# conversation-mode Specification

## ADDED Requirements

### Requirement: 系统必须提供独立于项目的对话作用域

系统 MUST 提供一个不绑定项目目录的“对话”作用域，并以用户主目录下的 `.willow` 作为默认根目录。

#### Scenario: 首次进入对话时自动创建默认根目录

- **前提** 用户首次使用“对话”功能
- **并且** 用户主目录下不存在 `.willow` 目录
- **当** 用户进入“对话”入口或发送第一条对话消息
- **那么** 系统自动创建 `.willow` 目录
- **并且** 将该目录作为对话作用域的默认根目录

#### Scenario: 对话作用域不绑定任何项目工作空间

- **前提** 用户当前进入的是“对话”功能
- **当** 系统构建 Agent 运行上下文
- **那么** 系统不得要求用户先选择某个项目工作空间
- **并且** Agent 的工作根目录使用 `.willow`

#### Scenario: 删除项目工作空间不影响对话作用域

- **前提** 系统中已存在若干项目工作空间和一个对话作用域
- **当** 用户删除某个项目工作空间
- **那么** 系统只删除该项目工作空间及其关联数据
- **并且** 对话作用域和其无限会话保持不变

### Requirement: 对话作用域必须表现为单条无限会话流

系统 MUST 将“对话”作用域呈现为持续追加的一条无限会话流，而不是项目式会话树。

#### Scenario: 进入对话页时获取或创建当前无限会话

- **前提** 用户点击侧边栏中的“对话”入口
- **当** renderer 请求当前对话内容
- **那么** 主进程返回对话作用域下现有的无限会话
- **并且** 若该会话不存在则自动创建一条新的空会话

#### Scenario: 后续消息持续追加到同一条对话流

- **前提** 用户已经处于“对话”模式
- **并且** 系统已经为该作用域创建当前无限会话
- **当** 用户继续发送消息
- **那么** 系统将消息追加到同一条对话会话
- **并且** 不要求用户手动新建下一条会话

#### Scenario: 对话入口不暴露历史会话切换

- **前提** 用户正在使用“对话”功能
- **当** 用户查看左侧导航
- **那么** 系统不展示该作用域下的历史会话列表
- **并且** 不展示“查看更多历史”或等价的会话树入口
