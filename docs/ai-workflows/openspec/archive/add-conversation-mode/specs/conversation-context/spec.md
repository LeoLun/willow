# conversation-context Specification

## ADDED Requirements

### Requirement: 对话作用域必须使用独立的上下文压缩机制

系统 MUST 为“对话”作用域使用独立于项目工作空间会话的上下文压缩机制，以适配无限对话流。

#### Scenario: 对话作用域触发独立压缩策略

- **前提** 当前发送发生在“对话”作用域
- **并且** 历史上下文接近模型安全阈值
- **当** 系统准备构建本轮 Agent 上下文
- **那么** 系统使用对话专属压缩策略
- **并且** 不直接沿用普通项目工作空间会话的单层摘要逻辑

#### Scenario: 项目工作空间会话保持现有压缩策略

- **前提** 当前发送发生在项目工作空间会话
- **当** 系统判断是否需要压缩
- **那么** 系统继续使用现有项目会话压缩逻辑
- **并且** 不进入对话专属的无限流压缩分支

### Requirement: 对话压缩必须保留长期稳定信息与未闭环事项

系统 MUST 在对话压缩结果中稳定保留长期偏好、关键约束和未完成事项，而不只是最近消息摘要。

#### Scenario: 长期偏好和约束被提炼进稳定层

- **前提** 早期对话中已经出现用户明确偏好、固定约束或长期工作方式
- **当** 系统执行对话压缩
- **那么** 系统把这些信息提炼为稳定上下文层
- **并且** 后续轮次构建上下文时持续注入该层信息

#### Scenario: 未闭环事项在压缩后仍可追踪

- **前提** 对话历史中存在尚未完成的问题、待办或待验证项
- **当** 系统执行对话压缩
- **那么** 系统将其归入未闭环事项层
- **并且** 后续发送时模型仍能读取这些未完成事项

### Requirement: 对话压缩必须防止摘要自身无限增长

系统 MUST 对对话压缩结果执行分层整理或再压缩，避免摘要本身随着无限对话持续膨胀。

#### Scenario: 滚动摘要层达到阈值时再次整理

- **前提** 对话作用域已经经历多次压缩
- **并且** 压缩产生的滚动摘要层再次逼近安全阈值
- **当** 系统准备本轮发送
- **那么** 系统对较早摘要进行再整理或高层合并
- **并且** 保持整体输入仍低于安全阈值

#### Scenario: 热窗口始终优先保留

- **前提** 当前对话正在进行，最近几轮消息直接影响本轮回答
- **当** 系统执行对话压缩
- **那么** 系统优先保留最近完整热窗口
- **并且** 不在同一轮用户请求与其直接回复之间任意截断
