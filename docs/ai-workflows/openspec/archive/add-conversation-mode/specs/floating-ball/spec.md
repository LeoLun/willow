# floating-ball Specification

## ADDED Requirements

### Requirement: 悬浮球动画只响应对话作用域

系统 MUST 让悬浮球动画只响应“对话”作用域的流式输出状态，而忽略项目工作空间会话的流式事件。

#### Scenario: 对话作用域开始流式输出时展示动画

- **前提** 当前存在一条属于“对话”作用域的消息发送
- **并且** Agent 开始流式输出
- **当** 悬浮球收到对应的开始事件
- **那么** 悬浮球进入动画状态

#### Scenario: 对话作用域流式结束时恢复空闲状态

- **前提** 悬浮球当前因“对话”作用域的流式输出处于动画状态
- **当** 同一条对话流收到结束事件
- **那么** 悬浮球退出动画状态并恢复空闲显示

#### Scenario: 项目工作空间会话不触发悬浮球动画

- **前提** 用户正在项目工作空间中进行聊天
- **并且** 该会话开始流式输出
- **当** 悬浮球收到该流式事件
- **那么** 悬浮球忽略该事件
- **并且** 不进入动画状态

### Requirement: 流式事件必须携带可判定的作用域信息

系统 MUST 在悬浮球消费的消息流式事件中携带足够的作用域标识，以便准确区分“对话”与“工作空间”。

#### Scenario: 事件中可识别是否属于对话作用域

- **前提** 主进程向 renderer 广播某次会话的流式事件
- **当** renderer 或悬浮球处理该事件
- **那么** 事件负载中包含 `chatScope`、`workspaceKind`、`isConversation` 或等价可判定字段
- **并且** 仅靠该字段即可判断是否应驱动悬浮球动画
