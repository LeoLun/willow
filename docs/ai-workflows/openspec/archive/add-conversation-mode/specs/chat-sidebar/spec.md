# chat-sidebar Specification

## ADDED Requirements

### Requirement: 侧边栏必须新增“对话”分区

系统 MUST 在左侧栏“工作空间”分区上方新增“对话”分区，作为一级导航入口。

#### Scenario: 对话分区位于工作空间上方

- **前提** 用户打开主界面左侧栏
- **当** 侧边栏完成渲染
- **那么** “对话”分区显示在“工作空间”分区之前
- **并且** 该分区在可视层级上被识别为一级导航，而不是某个项目子项

#### Scenario: 对话分区使用单入口而非树状结构

- **前提** 用户查看“对话”分区
- **当** 系统渲染该分区内容
- **那么** 系统只渲染一个平铺入口或等价的单入口容器
- **并且** 不显示折叠箭头、嵌套会话列表或历史下拉内容

#### Scenario: 点击对话入口进入对话页面

- **前提** 用户当前不在对话页面
- **当** 用户点击“对话”分区中的入口
- **那么** 系统导航到对话专用页面
- **并且** 该页面加载当前无限会话流

### Requirement: 新增对话分区不能破坏现有工作空间导航

系统 MUST 保持现有项目工作空间导航能力可用。

#### Scenario: 工作空间导航保留原行为

- **前提** 系统中已存在项目工作空间和若干项目会话
- **当** 用户展开或操作“工作空间”分区
- **那么** 现有项目树、项目会话列表和查看更多历史行为保持可用
- **并且** 新增“对话”分区不会吞并或替代项目工作空间分区
