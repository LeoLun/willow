# 设置页规格

## 新增需求

### 需求：设置页必须拆分为独立设置工作区

系统必须将设置页呈现为独立设置工作区，而不是将所有设置项堆叠在单一内容流中。

#### 场景：进入设置工作区

- **前提** 用户通过主侧边栏或其他泛化设置入口进入设置
- **当** 设置页面渲染
- **那么** 页面展示设置专属侧边栏和右侧内容区
- **并且** 设置专属侧边栏包含“返回应用”“外观”“配置”
- **并且** 页面不展示主应用左侧导航、主应用侧边栏触发按钮或工作空间列表
- **并且** 页面不会形成“主应用侧边栏 + 设置侧边栏 + 内容区”的三栏布局
- **并且** 页面默认展示“外观”内容

#### 场景：直接访问设置子页面

- **前提** 用户直接访问设置子页面 URL
- **当** URL 指向“外观”或“配置”
- **那么** 系统展示对应子页面内容
- **并且** 设置侧边栏中对应导航项处于选中态

#### 场景：从设置页返回应用

- **前提** 用户当前位于任一设置子页面
- **当** 用户点击“返回应用”
- **那么** 系统离开设置工作区
- **并且** 在存在可恢复来源时返回来源页面
- **并且** 在没有可恢复来源时回到主应用默认页面

### 需求：设置侧边栏必须只包含外观和配置两项

系统必须在设置页内部侧边栏中提供稳定的二级导航，且本次只包含“外观”和“配置”两项。

#### 场景：展示设置导航

- **前提** 用户位于设置工作区
- **当** 设置侧边栏渲染
- **那么** 导航项包含“外观”和“配置”
- **并且** “外观”使用能表达显示主题的图标
- **并且** “配置”使用能表达系统配置的图标
- **并且** 不展示本次范围外的其他设置分类

#### 场景：切换设置分区

- **前提** 用户位于设置工作区
- **当** 用户点击“外观”或“配置”
- **那么** 系统通过路由切换到对应设置子页面
- **并且** 当前选中态随路由更新
- **并且** 切换分区不会触发与目标分区无关的数据请求

### 需求：外观页必须承载主题模式设置

系统必须在外观页提供当前已有的主题模式设置能力。

#### 场景：展示外观页

- **前提** 用户进入外观设置页
- **当** 页面渲染
- **那么** 页面标题为“外观”
- **并且** 页面展示“外观”设置分组
- **并且** 分组说明为自定义应用显示主题的语义
- **并且** 页面展示“主题模式”字段

#### 场景：切换主题模式

- **前提** 用户位于外观设置页
- **当** 用户选择“跟随系统”“浅色”或“深色”
- **那么** 应用主题模式立即按所选值更新
- **并且** 当前选中的主题模式有明确选中态
- **并且** 行为与现有主题切换能力一致

### 需求：配置页必须承载模型与 Tavily Key 管理

系统必须在配置页保留当前已有的模型配置与 Tavily Key 管理能力。

#### 场景：展示配置页

- **前提** 用户进入配置设置页
- **当** 页面渲染
- **那么** 页面标题为“配置”
- **并且** 页面展示“模型配置”分组
- **并且** 页面展示“网络搜索 (Tavily)”分组
- **并且** 两个分组分别拥有标题、短说明、主体列表和主操作入口

#### 场景：管理模型配置

- **前提** 用户位于配置设置页
- **当** 模型列表可用
- **那么** 页面展示模型名称、默认状态、提供方、模型 ID 和 Base URL
- **并且** 用户可以添加模型、编辑模型、删除模型、设置默认模型
- **并且** 这些操作复用现有模型弹窗和配置 store 语义

#### 场景：模型配置为空

- **前提** 用户位于配置设置页
- **当** 模型列表为空
- **那么** 页面展示明确空状态
- **并且** 用户仍然可以通过主操作添加模型

#### 场景：管理 Tavily Key

- **前提** 用户位于配置设置页
- **当** Tavily Key 列表可用
- **那么** 页面展示脱敏后的 Key、当前月用量、月度上限和进度条
- **并且** 用户可以添加 Key、编辑 Key、删除 Key
- **并且** 这些操作复用现有 Tavily Key 弹窗和配置 store 语义

#### 场景：Tavily Key 为空

- **前提** 用户位于配置设置页
- **当** Tavily Key 列表为空
- **那么** 页面展示明确空状态
- **并且** 用户仍然可以通过主操作添加 Key

### 需求：设置页界面必须遵循项目设计规范和参考设计稿

系统必须按照仓库根目录 `DESIGN.md` 与 `ui/work.pen` 中设置页参考实现 renderer 界面。

#### 场景：以工作台设置页结构渲染

- **前提** 用户位于设置工作区
- **当** 页面渲染
- **那么** 页面采用左侧设置导航和右侧内容区的稳定两栏结构
- **并且** 右侧内容区使用受控宽度和紧凑垂直节奏
- **并且** 设置页左侧栏替代主应用左侧栏，而不是追加在主应用左侧栏右侧
- **并且** 页面不使用营销式 Hero、装饰性背景、重阴影或新主题 token

#### 场景：隐藏主应用侧边栏

- **前提** 用户从主应用侧边栏或发送器入口进入设置页
- **当** 设置工作区渲染
- **那么** 主应用侧边栏不再显示
- **并且** 用户只能看到设置专属侧边栏和设置内容区
- **并且** 用户可通过“返回应用”离开设置工作区

#### 场景：保持设置分组层级

- **前提** 用户位于任一设置子页面
- **当** 设置分组渲染
- **那么** 每个分组只承载一个主题
- **并且** 分组包含标题、短说明和字段主体
- **并且** 主操作按钮位置稳定，行内操作使用图标按钮和 Tooltip
