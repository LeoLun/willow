# 提案：设计并实现 Willow 欢迎首页

## 动机

当前 Willow 桌面应用在启动或没有历史会话数据时，默认会重定向到全局对话路由 `/conversation`。由于此时没有历史消息且未处于流式响应状态，整个右侧主显示区域（由 `Session.vue` 渲染）会呈现完全空白的视觉状态。这对于新用户或无历史数据的用户而言体验不够友好，缺乏明确的操作引导和应用功能介绍。

因此，我们需要设计并实现一个符合 Willow `DESIGN.md` 设计规范的、冷静、专注、工具型的**欢迎首页 (Welcome/Home View)**，作为空会话状态下的默认展示，提供清晰的引导和快捷操作。

## 目标

1. **优化空状态体验**：当会话中没有任何消息且不处于加载/流式状态时，展示精心设计的欢迎首页，避免空白页。
2. **提供快捷操作引导**：在首页上放置高频的下一步操作入口：
   - **开启新对话**（快速在输入框中对焦并提示用户）
   - **创建工作空间**（触发“新建工作空间”弹窗）
   - **配置模型设置**（跳转至设置页面以配置 DeepSeek API Key 等）
3. **展示核心功能介绍**：以紧凑的卡片或面板形式，简要说明 Willow 的三大核心功能：
   - **全局对话**：支持上下文管理的通用 AI 助手。
   - **工作空间 Agent**：绑定本地项目目录，自动加载上下文并执行代码级变更。
   - **任务自动化**：定时或按需触发复杂的 AI 自动化流。
4. **符合设计规范**：复用项目已有的 `Card`、`Button`、`Separator` 等 `@willow/shadcn` 组件和设计 token，呈现出沉稳、高级的桌面工作台质感，避免花哨的营销页风格。

## 范围

- **前端组件新增**：在 `app/work/src/renderer/src/pages/chat/session/components/` 下新建 `WelcomeView.vue`。
- **Session 页面调整**：修改 `Session.vue`，在消息列表为空且未在流式传输时，渲染 `WelcomeView.vue`。
- **交互与快捷操作对接**：
   - 快捷动作“创建工作空间”：调用已有的 `useDialog` 打开 `CreateWorkspace` 弹窗。
   - 快捷动作“去配置模型”：跳转至 `/setting/configuration`。
   - 快捷动作“开始对话”：在首页提供建议提示词卡片，点击后自动填入输入框，或使底部的输入框自动获取焦点。

## 非范围

- 本次变更不修改底层的路由结构（仍保留 `/conversation` 作为默认入口）。
- 不添加额外的营销插画或外部重型图片资源，全部使用 Lucide 图标和 Vanilla CSS 样式。
- 不影响现有的消息发送、流式接收和工作空间核心逻辑。
