# 欢迎首页功能规范 (Specification)

## 目的

定义欢迎首页 (Welcome View) 的展示逻辑、交互行为与状态控制，确保在会话无数据时呈现正确的界面引导。

## 需求 (Requirements)

### Requirement: 智能判断空状态展示欢迎页

系统 MUST 仅在会话中无历史消息且未发生数据传输时展示欢迎页。

#### Scenario: 默认新会话展示欢迎页
- **前提** 会话消息列表 `messages` 长度为 `0`
- **并且** 状态 `isStreaming` 为 `false`
- **当** 用户打开当前会话
- **那么** 系统渲染并展示欢迎页组件
- **并且** 隐藏消息历史列表

#### Scenario: 收到首条消息或流式启动时隐藏欢迎页
- **前提** 欢迎页当前处于展示状态
- **当** 用户发送了第一条消息，或者 `isStreaming` 变更为 `true`
- **那么** 系统的欢迎页 MUST 立即隐藏
- **并且** 系统立即渲染消息历史列表及流式接收容器

---

### Requirement: 快捷操作引导响应

系统提供多个快捷入口以快速完成核心配置与操作。

#### Scenario: 触发创建工作空间
- **前提** 用户处于欢迎页
- **当** 用户点击“创建项目工作空间”操作卡片
- **那么** 系统弹出“新建工作空间”对话框，方便用户选择本地文件夹

#### Scenario: 跳转到模型配置
- **前提** 用户处于欢迎页
- **当** 用户点击“配置 DeepSeek API Key”操作卡片
- **那么** 系统切换路由跳转到 `/setting/configuration` (模型配置页面)

---

### Requirement: 提示词建议点击填充与对焦

用户可以通过点击建议的提示词，快速输入内容并自动聚焦。

#### Scenario: 点击建议提示词填充编辑器并获取焦点
- **前提** 用户处于欢迎页，底部输入框为空
- **当** 用户点击任一“你可以试试问我”提示词卡片
- **那么** 系统自动将对应的提示词内容填充至底部的 Tiptap 编辑器中
- **并且** 编辑器自动获取输入焦点，光标处于提示词文本末尾
