# Proposal: merge-sender-slash-resource-picker

## Background

`@willow/sender` 目前已经具备两套输入器搜索能力：

- 输入 `/` 打开技能选择面板，并插入技能标签。
- 输入 `@` 打开工作空间文件选择面板，并插入文件标签。

这让“为当前消息选择上下文”的入口被拆成两套触发符和两个弹层。用户现在希望将两者统一：`/` 既能选择技能，也能选择文件；`@` 不再作为文件搜索入口；弹窗只保留一个统一资源选择器。

用户指定 UI 稿以 [`ui/work.pen`](/Users/liujinglun/code/willow/ui/work.pen) 中 `Willow / Sender Component States` 的 `03 输入 / 后效果` 为准。该稿展示了输入 `/` 后，输入器上方出现一个统一 command palette，内部按 `插件`、`技能`、`文件` 分组展示候选项；输入器本体仍是一体式 sender 容器，底部工具栏保留 `+`、`/`、联网、模型选择和发送按钮。

根据仓库根目录 [`DESIGN.md`](/Users/liujinglun/code/willow/DESIGN.md)，该交互应保持桌面工作台风格：紧凑、稳定、可扫读，使用现有 token、`@willow/shadcn` 与 lucide 图标，不新增独立主题或营销式说明。

## Goals

- 将 `/` 和 `@` 的搜索入口统一为 `/`。
- 输入 `/` 后打开一个统一弹层，在同一弹层中展示插件、技能、文件候选项。
- 取消 `@` 作为工作空间文件搜索触发符；`@` 作为普通文本输入保留。
- 选择技能后继续插入现有技能标签，并保持 `selectedSkills` 结构化发送语义。
- 选择文件后继续插入现有文件标签，并保持 `selectedFiles` 结构化发送语义。
- 统一弹层视觉还原 `work.pen` 中 `03 输入 / 后效果`：分组标题、紧凑行高、活动项浅底、单个浮层、一体式输入器。
- 保留左下 `+` 系统文件选择器入口，不把它改为工作空间文件搜索。

## Non-Goals

- 本次不实现新的技能安装、插件安装、文件内容读取或附件上传能力。
- 本次不改变系统文件选择器 `+` 的行为。
- 本次不改变技能标签、文件标签在消息中的 Markdown 文本格式。
- 本次不改变“仅文件标签且无正文不可发送”的既有发送规则，除非实现阶段发现当前行为已经不同。
- 本次不重做模型选择、联网开关、用量展示或发送 / 停止流程。

## User Experience

用户在 sender 编辑区输入 `/` 后，光标所在输入器上方打开统一资源选择弹层。弹层按照 `插件`、`技能`、`文件` 的顺序展示当前可用候选项；当某一类没有可用候选或不适用于当前宿主时，该分组可以隐藏，避免显示误导性的空区块。

用户继续输入 `/work`、`/app` 等查询时，弹层在所有可用分组中统一过滤。键盘上下键在可见结果之间移动活动项，Enter 选择活动项，Escape 关闭弹层。选择后，触发文本不会作为正文保留：技能转为技能标签，文件转为文件标签，插件转为对应的插件启用标识或宿主定义的结构化选择结果。

如果用户输入 `@`，sender 不再打开文件搜索面板；`@` 留在正文中，供用户正常输入账号、邮箱、自然语言或其他文本。

## Approaches

### Approach A: 保留 `/` 与 `@`，只把两个面板视觉做得更像

优点：

- 改动较小。
- 现有文件选择行为几乎不需要调整。

缺点：

- 不满足“统一合并为 `/`”的核心要求。
- 用户仍需要记住两个触发符。
- 两个弹层仍然存在，无法还原 `work.pen` 的统一 command palette。

### Approach B: 用 `/` 统一触发资源选择器，移除 `@` 文件搜索触发

优点：

- 直接满足用户要求。
- 技能、文件、插件共享一套触发、过滤、键盘导航和弹层位置。
- 与 `work.pen` 的 `03 输入 / 后效果` 一致。
- 能减少 sender 内部的双面板状态分支。

缺点：

- 需要调整 `useTriggerManager` 使用方式、面板组件和键盘选择逻辑。
- 需要迁移文件选择手动入口，使其不再依赖 `@` 触发符。

### Approach C: 引入新的 command palette 快捷键，同时保留 `/` 与 `@`

优点：

- 为未来命令系统留下更大空间。
- 可降低对既有触发符用户的影响。

缺点：

- 增加第三套入口，复杂度更高。
- 不符合用户指定的 `/` 统一入口。
- 不利于还原当前 UI 稿。

## Recommendation

推荐 Approach B。

本次需求明确要求把 `/` 技能选择与 `@` 文件选择合并到 `/`，且 UI 稿已经给出统一弹层结构。实现时应把“资源选择”抽象为 sender 内的一类统一候选项，由 `/` 触发并在一个 panel 中分组渲染；选中后再按候选项类型调用现有技能标签、文件标签或插件选择逻辑。

## Success Criteria

- 输入 `/` 后只出现一个统一资源选择弹层。
- 统一弹层中可同时展示插件、技能、文件分组，并按查询统一过滤。
- 输入 `@` 不会打开文件选择弹层。
- 技能选择仍插入技能标签，发送 payload 仍包含 `selectedSkills`。
- 文件选择仍插入文件标签，发送 payload 仍包含 `selectedFiles`。
- 左下 `+` 仍打开系统文件选择器，并复用文件标签插入效果。
- 弹层和输入器视觉与 `work.pen` 的 `03 输入 / 后效果` 保持一致。
- 现有模型选择、联网开关、发送 / 停止、用量展示不回退。
