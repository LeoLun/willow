# Design: add-chat-right-sidebar

## Overview

本次改动为聊天页增加一个可折叠的右侧辅助栏。它不是独立页面，而是聊天页壳层的一部分，用来承载当前上下文的次级信息与操作：

- 在 `workspace` 场景中承载工作空间设置
- 在 `session` 场景中承载当前会话概览、TODO 进度和工作空间文件列表

实现必须遵守仓库 [`DESIGN.md`](/Users/liujinglun/code/willow/DESIGN.md) 的约束：桌面工作台气质、稳定布局骨架、紧凑但可读的信息密度、克制的动效和清晰的主次层级。

## Chosen Approach

采用“聊天页壳层统一右栏 + 子路由提供上下文内容”的方案。

具体上：

- `Chat.vue` 负责页面整体三段式结构：标题区 / 主内容区 / 输入区
- 右侧栏作为和主内容并列的次级区块存在，由 `Chat.vue` 管理展开状态和布局变化
- 当前子路由为 `workspace` 时，右栏渲染工作空间设置面板
- 当前子路由为 `session` 时，右栏渲染会话信息面板

这样可以保证折叠入口始终位于聊天页右上角，而不是散落在不同子页面里。

## Layout Model

### Page Shell

聊天页应形成统一壳层：

- 顶部：当前主内容标题区域，且右上角提供右栏折叠 / 展开按钮
- 中部：左侧主内容区 + 右侧辅助栏
- 底部：消息发送区

当右栏展开时：

- 主内容区继续保持消息阅读和表单输入为主，不被右栏夺走视觉主导权
- 右栏宽度稳定，适合展示设置表单、概要卡片和文件树

当右栏折叠时：

- 主内容区回收横向空间
- 右栏保留最小化占位或完全收起，但按钮仍保持可发现
- 动画使用轻量宽度过渡，不使用夸张 motion

### Collapse Trigger

折叠按钮必须放在聊天页右上角，而不是右栏内部中段或输入区附近。

交互规则：

- 展开状态点击后收起右栏
- 折叠状态点击后展开右栏
- 按钮图标方向应清晰表达“向右栏收起 / 展开”的空间关系
- 切换时不应影响当前消息输入内容或滚动位置

## Route-Specific Right Sidebar Content

### Workspace Sidebar

`workspace` 路由下，右栏展示工作空间设置，信息结构参考 [`workspace-right-sidebar.tsx`](/Users/liujinglun/code/willow/docs/ui-design/workspace-right-sidebar.tsx)，但实现时需要收敛到 Willow 的桌面工作台风格。

至少包含：

- 工作空间基础信息
- 工作空间路径相关设置或展示
- `AGENTS.md` 相关设置区
- 保存操作区

约束：

- 面板是“设置页分组”语义，而不是营销式说明页
- 每组内容使用清晰标题、短说明和稳定字段间距
- 即便未来能力分阶段落地，右栏结构也要先固定为设置分组模式

### Session Sidebar

`session` 路由下，右栏展示当前会话信息，信息结构参考 [`chat-right-sidebar.tsx`](/Users/liujinglun/code/willow/docs/ui-design/chat-right-sidebar.tsx)。

右栏至少包含两个主标签：

- `概要`
- `文件`

#### 概要标签

概要标签包含两部分：

1. 会话基础信息
2. `TODO` 工具进度展示

会话基础信息至少应覆盖：

- 会话标题
- 创建时间或最近活跃时间中的可用关键信息
- 消息数量等可直接由当前会话历史推导的数据

`TODO` 展示迁移规则：

- 当前位于输入框上方的 `TodoProgress` 不再作为悬浮块显示
- `TODO` 进度展示移动到“概要”标签中的会话信息之后
- 展示应继续复用当前 `useTodoProgress` 的实时数据语义
- 若当前会话无 TODO 数据，则概要区应保持稳定留白或隐藏该分组，不制造空壳卡片

#### 文件标签

文件标签展示“当前工作空间的文件列表”，不是当前会话上传附件列表。

规则：

- 数据源必须来自当前会话所属 `workspace.path`
- 列表以树状层级展示文件和目录
- 允许目录展开 / 收起
- 文件图标可以按扩展名做轻量区分，但不能喧宾夺主
- 当工作空间路径无效、不可读或为空时，需要有明确受控状态

本次只要求文件列表浏览，不要求：

- 文件预览
- 文件搜索
- 文件编辑
- 拖拽上传或重命名

## Data Model And Loading

### Existing Reusable Data

当前仓库已经具备以下能力，可直接复用：

- 工作空间列表与单个工作空间信息
- 会话列表基础数据
- 会话历史消息
- 会话进行中的 `TODO` 实时数据

### New Data Needed

为了支撑右栏文件标签与更完整概要，需要新增或明确以下读取能力：

- 根据 `workspaceId` 或 `workspace.path` 获取当前工作空间文件树
- 为当前会话概要提供稳定的派生信息，如消息数、时间信息

文件树能力约束：

- 仅面向当前工作空间根目录
- 默认需要排除明显无意义或高噪声目录时，应在实现阶段明确规则
- 若目录层级很深，允许分层懒加载或一次读取后按树渲染，但行为必须对用户稳定

### State Ownership

建议状态划分如下：

- `Chat.vue`：右栏是否展开、当前右栏类型、整体布局状态
- `Workspace` 相关面板：工作空间设置数据与保存交互
- `Session` 相关面板：会话概览数据、TODO 数据、文件树数据

这样可以避免：

- 由子页面重复管理折叠状态
- 右栏 UI 和数据请求边界混乱

## Visual Constraints

参考设计稿时必须遵守 `DESIGN.md`，重点包括：

- 右栏使用轻边框和稳定表面，不使用厚重阴影
- 标签页、分组标题、文件树和设置字段保持紧凑密度
- 主内容区永远是聊天页第一视觉主体，右栏是辅助信息区
- 不照搬设计稿里的演示性留白，要按真实业务密度收敛

## Interaction Rules

### Session Scroll And Input Stability

右栏展开 / 折叠不应造成以下副作用：

- 消息滚动位置异常跳动
- 输入框内容丢失
- 正在进行中的流式消息被打断

### Route Switching

当用户在 `workspace` 和 `session` 路由之间切换时：

- 右栏组件内容跟随上下文切换
- 右栏展开状态应保持可预期，一般不因子路由切换而强制重置
- 若目标路由缺少对应数据，展示受控空态或占位，而不是空白面板

### Empty And Error States

需要覆盖以下状态：

- `workspace` 信息尚未加载
- 当前会话不存在或关联工作空间缺失
- 工作空间文件读取失败
- 工作空间目录为空
- 当前会话没有 TODO 数据

这些状态都必须用简短、可理解的文案说明下一步，而不是直接展示技术错误。

## Resolved Decisions

### TODO 是否继续保留在输入区上方

不保留。`TODO` 展示迁移到 `session` 右栏概要区，聊天输入区恢复为更纯粹的输入职责。

### 文件标签是否显示当前会话上传文件

不显示。文件标签以当前工作空间文件树为准。

### 折叠按钮是否放在右栏内部

不放在右栏内部。按钮固定在聊天页右上角，由壳层统一管理。

### Workspace 与 Session 是否共享同一右栏骨架

共享同一壳层骨架，但内容模块分开定义。这样既统一布局，又保留不同上下文的语义差异。
