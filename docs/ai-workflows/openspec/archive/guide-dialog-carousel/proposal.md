# OpenSpec 变更：使用配置指引改为弹窗分页轮播

## 动机

当前“配置与使用指引”（`GuideDialog`）在一个垂直列表中同时展示了“配置模型密钥”、“绑定项目工作空间”和“后台自动化工作流”三个步骤。这种长列表布局在较小窗口下需要滚动，且信息一次性全部呈现在视觉上显得有些拥挤和繁杂。

将指引改为弹窗内一个页面只展示一个功能/配置的轮播（Step-by-step）形式，可以：
1. 聚焦用户的注意力，一次只理解或操作一个步骤；
2. 减少弹窗高度，使界面更加精致、冷静；
3. 增强引导体验，使用更符合工作台风格的“上一页 / 下一页”切换流；
4. 先在 `app/ui-playground` 独立验证交互效果，保证主应用集成时的稳定性。

## 目标

1. 将使用配置指引（GuideDialog）改造成单页面步骤轮播结构；
2. 提供“上一页”和“下一页”导航切换，自动管理步骤索引状态；
3. 当到达最后一个步骤（自动化工作流）时，右下角的“下一页”文案和行为自动变为“确定”（关闭弹窗）；
4. 首选在 `app/ui-playground`（UI Playground）中实现该指引的重构与功能验证；
5. 在 UI Playground 验证无误后，将新设计同步应用至 `app/work` 主应用的 `GuideDialog.vue`。

## 范围

- 重构 `GuideDialog.vue`，实现 `currentStep` 状态切换。
- 对每个步骤使用独立的视觉面板，展示图标、步骤标题、描述性文本及操作按钮。
- 配置导航区，在底部页脚提供“上一页”与“下一页 / 确定”按钮，并展示精致的进度指示点（Indicators）。
- 保证 UI Playground 中调试欢迎首页时，该配置指引弹窗的全部跳转事件（配置 API Key、创建工作空间、确定）触发正常并有日志输出。
- 将重构后的弹窗整合进 `app/work` 并通过 `pnpm lint` 等代码检查。

## 非范围

- 不改变欢迎首页 `WelcomeView` 的触发机制和布局。
- 不修改具体的设置页（`/setting/configuration`）或创建工作空间对话框的业务逻辑本身。
