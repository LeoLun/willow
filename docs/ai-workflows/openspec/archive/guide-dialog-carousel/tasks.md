# 任务列表：使用配置指引改为弹窗分页轮播

## 1. 独立环境开发验证 (UI Playground)
- [ ] 备份或重写 `app/work/src/renderer/src/pages/chat/session/components/GuideDialog.vue`。
  - 由于 UI Playground 会引用这个组件，我们直接在该组件中进行分页逻辑的实现。
- [ ] 在 `GuideDialog.vue` 中定义步骤数据：
  - `steps` 数组，包含三页数据（模型配置、工作空间、自动化流）。
  - `currentStep` 响应式状态，初始为 0。
- [ ] 编写弹窗内部布局结构：
  - 顶部：标题（显示当前步骤指示，如 `配置与使用指引 (1/3)`）。
  - 中部：内容卡片区域，带平滑的过渡效果，展示图标、步骤标题、描述文本，以及相应的操作按钮（仅在步骤有 Action 时展示）。
  - 底部：指示器小圆点 + 上一页按钮 + 下一页（或确定）按钮。
- [ ] 启动 UI Playground 并手动验证：
  - 点击“配置与使用指引”拉起弹窗，确认默认显示第一页。
  - 检查第一页无“上一页”按钮，只有“下一页”。
  - 点击“下一页”，无缝过渡到第二页；此时显示“上一页”和“下一页”。
  - 点击“下一页”，过渡到第三页；此时“下一页”文案变为“确定”。
  - 点击“上一页”能顺利退回。
  - 在步骤一点击“配置 API Key”，验证弹窗正确关闭，且 Playground 日志面板记录了跳转到设置的事件。
  - 在步骤二点击“创建工作空间”，验证弹窗正确关闭，且 Playground 日志面板记录了创建工作空间的事件。
  - 在步骤三点击“确定”，验证弹窗正常关闭。

## 2. 主应用端集成验证 (Willow Electron App)
- [ ] 在 `app/work` 下启动开发服务器：`pnpm dev`。
  - 手动验证：在主应用空会话界面拉起该弹窗，检查各个分页的内容和切换是否完全正常。
  - 测试各步骤中业务动作按钮的点击跳转逻辑：
    - 点击“配置 API Key”是否能顺利关闭弹窗并跳转至设置页。
    - 点击“创建工作空间”是否能顺利关闭弹窗并弹出创建工作区对话框。
    - 点击“确定”能否正常关闭弹窗。

## 3. 代码质量与格式化
- [ ] 运行 `pnpm lint` 确保没有 Oxlint 警告或错误。
- [ ] 运行 `pnpm build` 确认项目构建不会出错。
