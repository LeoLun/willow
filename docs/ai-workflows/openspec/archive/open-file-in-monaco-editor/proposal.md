# Proposal: 侧边栏的文件管理支持使用 monaco-editor 打开文件（只读模式）

## 动机

当前 Willow 的右侧边栏（工作空间/会话详情）具备了“文件”选项卡，以树状结构展示了当前工作空间的文件树。但是，目前点击文件并不能在应用中直接预览其内容，用户只能使用系统关联程序打开。
为了提升工作空间的内聚性，使用户能够在 Willow 内快速、安全地浏览文件内容，我们计划引入 Monaco Editor。在侧边栏的文件树中，当用户点击某个文件时，弹出一个只读的 Monaco Editor 编辑器，以提供高性能、语法高亮和优雅的代码阅读体验。

## 目标

1. **内嵌只读预览**：在文件管理树中点击文件时，能以弹窗（Dialog/Modal）形式展示文件内容。
2. **支持语法高亮**：根据文件扩展名，自动匹配 Monaco Editor 的语言模式，进行语法高亮展示。
3. **安全与只读**：文件预览默认且强制处于 `readOnly: true` 只读状态，防止误触修改。
4. **极致性能**：借助 Monaco Editor 实现对大文件的流畅滚动和渲染。

## 成功标准

1. 用户可以在侧边栏文件树中点击任意文件。
2. 点击文件后，弹出一个弹窗展示文件内容。
3. 弹窗中的编辑器使用 Monaco Editor 渲染，不支持文本编辑（只读），但支持文本选择和复制。
4. 编辑器根据文件类型（`.ts`, `.js`, `.json`, `.vue`, `.md` 等）正确提供语法高亮。
5. 弹窗有关闭按钮或点击外部遮罩可关闭。
