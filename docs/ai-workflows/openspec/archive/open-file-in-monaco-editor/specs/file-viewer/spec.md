# Monaco Editor 文件预览规范

## 需求

### R1: 内联文件预览双栏布局与交互

- “文件”标签页的内容区 MUST 采用左右两栏排版（双栏布局）：
  - 左侧部分：渲染工作空间文件树，支持纵向独立滚动，固定宽度为如 `240px`。
  - 右侧部分：文件内容展示区，自适应填满剩余空间，内部渲染 Monaco Editor。
- 用户在左侧文件树中点击任何一个文件节点，右侧展示区 MUST 动态加载该文件，并在 Monaco Editor 中内联展示其内容。
- 当未选择任何文件时，右侧展示区应显示一个友好提示（如“选择一个文件以预览”）。
- 用户切换到其他标签页（如“设置”、“应用”）或切换工作空间时，应当清除当前选择的文件状态。

### R2: 只读 Monaco Editor 配置

- Monaco Editor 实例初始化时，MUST 配置以下属性以保证只读与阅读体验：
  - `readOnly: true` (禁止修改内容)
  - `domReadOnly: true` (在 DOM 层面辅助禁止输入)
  - `minimap: { enabled: true }` (启用右侧缩略图以方便大文件定位)
  - `theme: 'vs-dark'` 或根据系统当前主题自动选择适配的主题 (暗色/亮色)
  - `automaticLayout: true` (确保容器尺寸改变时编辑器能自动重绘布局)
  - `wordWrap: 'on'` (默认开启自动换行)

### R3: 文件高亮识别逻辑

- 编辑器 MUST 根据文件的扩展名，推断并设置 Monaco Editor 的 `language` 属性：
  - `.js`, `.mjs`, `.cjs` -> `javascript`
  - `.ts`, `.mts`, `.cts` -> `typescript`
  - `.json` -> `json`
  - `.md` -> `markdown`
  - `.vue` -> `html` (或者 `xml`，在 Monaco 没有内置 `vue` 语言解析器时回退到 `html`)
  - `.css` -> `css`
  - `.html` -> `html`
  - `.py` -> `python`
  - 其他后缀默认 -> `plaintext`

### R4: IPC 安全文件读取限制

- 主进程的 `READ_FILE` IPC 处理器在接收到路径后，MUST 验证目标路径是否位于当前激活的工作空间根目录下，防止非法路径穿越读取系统敏感文件。
- 如果文件过大（如超过 10MB）或为二进制格式，应给出相应友好提示，或者仅读取前 N 个字符。

## 验收标准

- [ ] 在侧边栏“文件”标签页左侧的文件树点击 `.ts` 或 `.json` 文件，右侧的内联 Monaco Editor 能显示对应文件的内容，并高亮展示。
- [ ] 尝试在内联编辑器中输入、退格或删除，编辑器没有任何反应且不触发任何修改，选中文本进行复制功能正常。
- [ ] 切换工作空间后，重置右侧预览状态。
- [ ] 无法通过非法路径传参（如 `../../../../etc/passwd`）来读取工作空间外的敏感文件。
