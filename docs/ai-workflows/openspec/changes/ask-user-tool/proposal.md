# 补充 ask_user 工具

## 动机

在 AI 协作执行复杂任务的过程中，AI 经常需要向执行者（用户）请求补充信息，或者让用户在若干选项中进行选择（例如：生成周报需要哪些素材、使用何种实现方案等）。
当前系统仅支持“权限申请”（Permission Approval）这一种交互，无法支持 AI 主动向用户发起结构化提问并获取回答的场景。

为了让 AI 在遇到模糊需求时可以主动向用户澄清，我们需要补充 `ask_user` 工具，允许 AI 主动要求补充信息。

## 目标

1. 新增 `ask_user` 工具，供 AI 遇到模糊任务或需要用户决策时调用。
2. 支持 AI 提供 2～4 个候选项供用户直接选择。
3. 支持用户手动输入其他回答。
4. 支持用户跳过或关闭提问。
5. 整体 UI 参考权限申请工具样式，设计精美且具有良好的微交互。

## 范围

- **工具定义**：在 `packages/core` 中实现 `ask_user` 工具，接受 `question` 和 `options` 两个参数。
- **通信层扩展**：修改 `ToolApprovalCoordinator`，支持在审批通过时传递具体原因（用户的回答或选项内容）。
- **组件实现**：将原预留骨架 `ArkUserPanel.vue` 重命名并完善为 `AskUserPanel.vue`，实现像素级还原的提问交互界面。
- **页面集成**：在 `Chat.vue` 中集成 `AskUserPanel`。当有 `ask_user` 工具的待审批任务时，在输入框上方/相同位置弹出该面板。
- **返回机制**：当用户选择某个选项或输入内容并确认时，将用户的输入作为工具执行的结果返回给 AI；当用户点击跳过或关闭时，返回“用户跳过问题”以使 AI 继续执行其他逻辑。

## 非范围

- 本次修改不涉及为其它工具添加选项功能，仅针对 `ask_user` 自研工具进行特殊 UI 渲染。
- 不影响正常的非提问类工具的权限申请流程。
