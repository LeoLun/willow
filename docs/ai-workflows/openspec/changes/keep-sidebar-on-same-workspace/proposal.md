# 提案：切换或新建会话时保持相同工作空间的右侧侧边栏数据不刷新

## 动机

在 Willow 桌面应用中，当用户切换会话（Session）或在当前工作空间下新建会话时，由于路由切换是异步获取新会话详情的过程，当前选中的 `workspaceId` 会短暂地变为 `0`（代表空/加载中），然后再恢复为正确的工作空间 ID。这会导致右侧侧边栏（`ChatRightSidebar.vue`）中的工作空间文件列表、文件预览、设置项及 MCP 服务器等数据被重置并重新读取，给用户带来不必要的闪烁和状态丢失（例如展开的文件树折叠、打开的预览文件关闭等）。

为了提升使用流畅度，如果切换/新建会话前后属于同一个工作空间，侧边栏的工作空间相关数据和状态应该保持不变，不进行重新加载或刷新。

## 目标

1. **稳定工作空间 ID (Stable Workspace ID)**：在右侧侧边栏组件中，建立一个过滤掉 `0`（以及无效 ID）的稳定工作空间 ID 引用。
2. **状态保持**：当在同一个工作空间内切换会话或新建会话时，侧边栏的文件树展开状态、选中的预览文件、设置内容、MCP 服务列表等均保持不变。
3. **正确切换**：当真正切换到不同工作空间下的会话或工作空间时，侧边栏能够正确且干净地刷新并加载新工作空间的数据。
4. **会话信息更新**：在侧边栏的“概要”标签页下，属于会话自身的信息（标题、创建时间、最近活跃、消息数量）随会话切换正常更新。

## 范围

- 右侧侧边栏组件：`app/work/src/renderer/src/pages/chat/components/ChatRightSidebar.vue`

## 非范围

- 导航栏或主聊天区域的数据流更新逻辑。
