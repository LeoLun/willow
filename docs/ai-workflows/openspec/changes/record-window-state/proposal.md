# 提案：记录并恢复主窗口的位置和大小

## 动机

当前应用启动时，主窗口（`MainWindow`）的大小和位置固定为默认的 `1200x800`，且每次打开都会重置。为了提升用户体验，应当自动记录用户关闭或调整窗口时的大小、位置及最大化状态，并在下一次启动时予以恢复。

## 目标

1. **自动记录状态**：当用户调整主窗口大小（`resize`）或移动主窗口位置（`move`）时，自动将当前的位置（`x`、`y`）、大小（`width`、`height`）及最大化状态（`isMaximized`）保存到本地的 `window-state.json` 文件中。
2. **防抖保存**：采用 500ms 的防抖机制，避免频繁拖拽或缩放窗口时造成过多的磁盘写操作。
3. **启动时恢复**：
   - 启动导入主窗口配置时，读取本地保存的 `window-state.json` 中的 `width`、`height`、`x`、`y`，并将其作为 `BrowserWindow` 的初始配置，避免窗口显示时的闪烁或二次调整。
   - 在窗口初始化后（`onInit`），若上次状态为最大化，则调用 `win.maximize()` 恢复最大化状态。
4. **安全处理**：若配置文件损坏或加载失败，应退回到默认的 `1200x800` 居中显示。

## 范围

- 主窗口类 `app/work/src/main/window/main.window.ts`。
- 本地存储路径：`app.getPath("userData")/window-state.json`。

## 非范围

- 其他子窗口或悬浮球（悬浮球已有其独立的 `floating-ball.json` 位置记录逻辑）。
