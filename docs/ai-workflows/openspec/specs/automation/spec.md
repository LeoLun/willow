# automation Specification

## Purpose
Define automation execution semantics, including how running records are treated and how short-lived in-process duplicate-run protection behaves.

## Requirements
### Requirement: 自动化执行不得被历史 running 记录阻止

系统 MUST NOT 把数据库中已有的 `automation_runs.status = "running"` 记录作为新的自动化执行门禁。

#### Scenario: 历史 running 记录不阻止立即执行

- **前提** 某条自动化存在一条历史运行记录
- **并且** 该历史运行记录的状态为 `running`
- **当** 用户在自动化详情页点击“立即执行”
- **那么** 系统仍然启动一次新的自动化执行
- **并且** 系统为本次执行创建新的会话
- **并且** 系统不因为该历史运行记录返回“自动化正在执行中”

#### Scenario: 应用停止后可以再次执行

- **前提** 某条自动化在执行过程中应用进程被停止
- **并且** 上一次运行记录可能仍保留为 `running`
- **当** 用户重新打开应用并再次立即执行同一条自动化
- **那么** 系统允许本次执行启动
- **并且** 不要求用户手动修改数据库或删除历史运行记录

#### Scenario: 定时执行不受历史 running 记录影响

- **前提** 某条启用的定时自动化存在历史 `running` 运行记录
- **当** cron 到达下一次计划触发时间
- **那么** 系统仍然尝试启动本次定时执行
- **并且** 不因为历史运行记录跳过本次计划

#### Scenario: 漏跑补偿不受历史 running 记录影响

- **前提** 某条启用的定时自动化存在历史 `running` 运行记录
- **并且** 应用启动时检测到需要补偿的最近一次漏跑
- **当** 系统执行漏跑补偿
- **那么** 系统仍然尝试启动本次补偿执行
- **并且** 不因为历史运行记录跳过补偿

### Requirement: 运行记录状态仅用于展示和结果记录

系统 MUST 允许继续记录自动化运行状态，但该状态不得表示跨进程执行锁。

#### Scenario: 新执行仍记录结果

- **前提** 用户立即执行某条自动化
- **当** 系统启动本次自动化执行
- **那么** 系统创建本次运行记录
- **并且** 会话创建后将运行记录关联到该会话
- **并且** 执行成功后将本次运行记录更新为 `completed`
- **并且** 执行失败后将本次运行记录更新为 `failed` 并记录错误信息

#### Scenario: running 只表示最后已知状态

- **前提** 某条运行记录的状态为 `running`
- **当** 系统判断新的自动化执行是否可以启动
- **那么** 该状态只被视为历史记录的最后已知状态
- **并且** 不作为拒绝新执行的依据

### Requirement: 进程内重复触发保护不得跨重启保留

系统 MUST 允许在当前进程生命周期内避免同一自动化被快速重复启动，但该保护不得持久化到数据库。

#### Scenario: 当前进程内重复触发可被短期拦截

- **前提** 当前应用进程中同一条自动化已经启动且尚未释放执行保护
- **当** 用户再次请求立即执行同一条自动化
- **那么** 系统可以拒绝本次重复请求
- **并且** 错误只表示当前进程内的短期执行冲突

#### Scenario: 重启后执行保护消失

- **前提** 当前进程内的自动化执行保护曾经存在
- **当** 应用进程退出并重新启动
- **那么** 该执行保护不再存在
- **并且** 系统不得从 `automation_runs` 表恢复该保护
