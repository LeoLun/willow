# 执行计划：使用配置指引改为弹窗分页轮播

将“配置与使用指引”（`GuideDialog.vue`）重构为步骤分页轮播模式，并在 UI Playground 以及主应用中验证其可用性。

## 任务拆解与执行步骤

### 1. 修改 GuideDialog.vue 实现步骤分页
- **目标**：将原有列表结构改造为单页卡片轮播结构，带切换导航。
- **文件**：`app/work/src/renderer/src/pages/chat/session/components/GuideDialog.vue`
- **步骤**：
  - 定义步骤数组：包含 3 个步骤（API Key 配置、工作空间绑定、自动化配置），每个步骤定义图标、标题、介绍文本和相关的跳转事件。
  - 添加 `currentStep = ref(0)` 来跟踪当前的页码（0 至 2）。
  - 更新 DialogContent 的高度和结构：
    - 主体展示区：使用简单的过渡效果展示当前步骤。
    - 底部展示区：
      - 左侧放置 3 个小指示圆点，当前步骤对应的圆点高亮。
      - 右侧放置“上一页”与“下一页 / 确定”按钮。
  - 点击“上一页”触发 `currentStep.value--`；点击“下一页”触发 `currentStep.value++`；点击“确定”或任意步骤中的跳转操作按钮（例如“配置 API Key”）触发关闭和外部逻辑。

### 2. UI Playground 验证
- **目标**：确保在 UI Playground 独立运行时，`GuideDialog.vue` 可以正常渲染，切换行为流畅，事件正常向上传递。
- **验证命令**：
  - 启动 UI Playground 运行环境：`pnpm dev:ui` (或在 `app/ui-playground` 路径下执行 `pnpm dev`)
  - 打开调试页面，点击“配置与使用指引”按钮，手动验证：
    - 默认显示第一步，底部没有“上一页”按钮。
    - 点击“下一页”进入第二步，显示“上一页”和“下一页”按钮。
    - 点击“下一页”进入第三步，按钮变为“确定”。
    - 分别点击动作按钮“配置 API Key”与“创建工作空间”，确认事件触发被 Playground 日志成功记录。

### 3. 主应用端集成验证
- **目标**：在 Electron 主应用环境下确认无报错，功能运行正常。
- **步骤**：
  - 启动主应用开发环境：`pnpm dev`
  - 进入空白会话界面，点击“配置与使用指引”呼出弹窗，验证行为一致性。
  - 检查各业务动作按钮触发后，设置页跳转和工作空间创建弹窗拉起均正常工作。

## 验证与测试步骤

### 自动化校验
- 运行 `pnpm lint` 检查代码格式与 lint。
- 运行 `pnpm build` 检查打包构建是否正常。

### 手动验收标准
1. **分页导航**：第一页没有“上一页”按钮；最后一页右下角按钮显示“确定”；中途可以往返切换。
2. **指示器同步**：指示小圆点数量和选中态与当前页码完全同步。
3. **跳转事件**：按钮“配置 API Key”与“创建工作空间”点击后，弹窗必须自动关闭，并成功触发其绑定的对应行为或事件。
